# Сайт подолога

Рабочий MVP сайта врача: фронт, запись пациента, SQLite-база, API и простая админка для просмотра заявок.

## Как запустить локально

В PowerShell из этой папки:

```powershell
node server.mjs
```

Открыть:

- сайт: `http://127.0.0.1:4173/`
- кабинет врача: `http://127.0.0.1:4173/admin.html`

Токен админки по умолчанию:

```text
dev-admin
```

Если PowerShell не дает запускать `npm`, можно пользоваться `node` напрямую. Или запускать npm как `npm.cmd run smoke`.

## Проверка

```powershell
node --check server.mjs
node scripts/smoke-test.mjs
```

Smoke-test проверяет полный сценарий: услуги, слоты, создание записи, список в админке, смена статуса.

## Что уже работает

- услуги хранятся в SQLite;
- реальные данные сайта вынесены в `config/site.json`;
- свободные окна создаются автоматически;
- пациент выбирает слот и отправляет заявку;
- форма требует согласие на обработку персональных данных;
- занятый слот больше не доступен для новой записи;
- врач видит заявки в админке;
- врач может менять статус: новая, подтверждена, выполнена, отменена.

## Что нужно сделать перед реальным запуском

1. Заменить имя, телефон, адрес, соцсети и тексты под реального врача.
   Основное место для правок: `config/site.json`.
2. Задать секретный админ-токен:

```powershell
$env:ADMIN_TOKEN = "сложный-секретный-токен"
node server.mjs
```

3. Подключить реальные уведомления врачу: Telegram-бот, WhatsApp-шлюз или письмо на почту.
4. Добавить политику конфиденциальности и согласие на обработку персональных данных.
5. Для продакшена поставить приложение на VPS или PaaS, подключить домен и HTTPS.
6. Если записей станет много, перенести базу с локального SQLite на PostgreSQL.
7. Подробный план запуска лежит в `docs/launch-plan.md`.

## Структура

- `index.html` - публичный сайт и форма записи;
- `admin.html` - кабинет врача;
- `privacy.html` - политика конфиденциальности;
- `consent.html` - согласие на обработку персональных данных;
- `config/site.json` - данные врача, услуги, цены, график;
- `server.mjs` - сервер, API, SQLite-схема и стартовые данные;
- `data/clinic.sqlite` - база появляется после первого запуска;
- `scripts/smoke-test.mjs` - быстрая проверка рабочего сценария.
- `docs/launch-plan.md` - что делать для домена, хостинга и запуска.
